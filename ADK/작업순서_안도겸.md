# 안도겸 개인 작업 순서 로드맵

> 참조: `파일별_역할분담.md`, `RAG_아키텍처_설계.md`, 기획안 v1.2
> 기준: ① 중요도(막는 정도) ② 프로젝트 이해도 기여 ③ 작성 난이도
> 원칙: **"일단 뭔가 돌아가는 걸 빨리 보는 것"이 최우선**

---

## 순서 설계 원칙

- **이해도 최우선**: 데이터 모양(`types.py`)과 노드 간 흐름(`state.py`)을 먼저. 이걸 쓰다 보면 전체 시스템이 머리에 그려짐.
- **중요도는 "막는 정도"**: 블로커 먼저, 있으면 좋은 것 뒤로. 평가·벤치마크는 최후.
- **쉬움 기준은 "보일러플레이트 먼저"**: 어댑터·래퍼류 먼저, 프롬프트 튜닝·캐시 임계값은 나중.

## Phase 0 — 시작 세팅 (반나절)

**목표**: `pip install` 하고 `.env` 세팅하면 import 에러 안 나는 상태.

| 순서 | 파일 | 담당 | 왜 지금 |
| --- | --- | --- | --- |
| 1 | `pyproject.toml` | 미정 → 안도겸 | 라이브러리 목록. `langchain`, `langgraph`, `fastapi`, `pydantic-settings`, `supabase`, `google-genai` 정도만 일단 |
| 2 | `config/settings.py` | 미정 → 안도겸 | BaseSettings 공식 문서 복붙. `GEMINI_API_KEY`, `SUPABASE_URL` 2~3 필드만 |
| 3 | `.env.example` | 안도겸 | settings.py와 짝. 본인 `.env` 만들기 |
| 4 | `README.md` | 안도겸 | "프로젝트 이름 + 실행 3줄"만. 나머진 나중 |

**끝났을 때**: `python -c "from config.settings import settings"` 에러 없음.

## 초보자 함정 3가지

### (1) "완벽하게 만들고 다음 단계" 금지
`mvp_chain.py`를 빨리 돌려보는 게 최우선. 거기서 깨닫는 게 많음. 리팩토링은 돌아간 다음에.

### (2) Phase 2 전까지 검색 품질 신경 X
retriever는 하드코딩이나 fake로. "LLM이 뭐라도 뱉는" 경험 먼저, 검색 품질은 Phase 3부터.

### (3) 프롬프트는 몰아서
왔다갔다 튜닝 금지. Phase 3의 17번(프롬프트 5개) 하루 집중해서 초안 쭉 뽑고, eval로 한꺼번에 개선.

## 체크포인트

각 Phase 끝에 "이것만 돌아가면 OK" 최소 기준:

- **Phase 0**: `from config.settings import settings` 에러 없음
- **Phase 1**: types.py의 Movie 객체 생성 가능, state.py의 QueryState TypedDict import 가능
- **Phase 2**: `mvp_chain.py` 실행 시 Gemini 답변 출력 ⭐
- **Phase 3**: 그래프에 질문 넣으면 5종 분류 중 하나로 라우팅되고 답변 생성
- **Phase 4**: 같은 질문 두 번째는 캐시 히트, 멀티턴에서 "그 영화" 문맥 유지
- **Phase 5**: `run_eval.py` 돌리면 50문항 정답률 숫자 출력
